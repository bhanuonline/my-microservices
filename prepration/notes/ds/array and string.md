Note :
* Use index when you also need to know where the maximum element is located.
* Use value when you only need the maximum number.
* PriorityQueue :Always get the highest-priority (smallest or largest) element quickly.Unlike a normal queue (FIFO), a PriorityQueue does not return elements in the order they were inserted.
  minHeap.peek(); // smallest element
  minHeap.poll(); // removes smallest element
  add() → throws an exception if insertion fails.
  offer() → returns false if insertion fails.
  heap only cares that the smallest element is at the top.
  Repeated poll(): Elements come out sorted ascending
  Heap Property is maintained
  Not guaranteed to be sorted
* Brute Force → Nested loops (O(n²))
  Better → Sort + Count (O(n log n))
  Best → HashMap (O(n))
