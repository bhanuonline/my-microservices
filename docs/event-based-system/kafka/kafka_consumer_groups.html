Kafka: consumer groups

Four interactive tabs — single group, scaling, multiple groups, and lag & rebalancing.

A consumer group is a set of consumers that cooperate to consume a topic. Kafka assigns each partition to exactly one member — so work is split, never duplicated within the group.

3 consumers, 3 partitions — a perfect split. Each partition is owned by exactly one consumer. No two consumers ever read the same message within the same group.
You can add or remove consumers at any time. Kafka rebalances — redistributing partitions across the current members. More consumers = more parallel throughput, up to the partition count.

Each group tracks its own offsets independently. fulfillment-group processing at its own pace has zero effect on analytics-group or notifications-group. One slow group never blocks another.
Lag = messages produced but not yet consumed. It's the most important operational metric for a consumer group. Rebalance = Kafka redistributing partitions after a consumer joins or leaves.
