Kafka Connect is a standalone runtime — you deploy connectors as JSON configs, not code. It handles retries, parallelism, offset tracking, and restarts automatically. Source connectors pull data into Kafka; sink connectors push data out.

The JDBC Source connector runs a SELECT query on an interval, tracks the highest offset column (e.g. updated_at or id) it has seen, and only fetches new rows next time. No code — just a JSON config POSTed to the Connect REST API.

The sink connector subscribes to the topic as a consumer group, batches records, and writes them to Elasticsearch. It commits Kafka offsets only after the batch is successfully indexed — so a failed ES write automatically retries without losing data.
